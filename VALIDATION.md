# Kairo Code — Dogfood 验证指南

感谢试用。这一版我们想验证一个核心命题：**当 AI（Crew）改完一堆代码后，你能不能在 60 秒内重新“看懂并信任”这个系统**——靠 Comprehension Gate（理解力闸门）+ 同屏的 Code System Map，而不是逐行重读 diff。

请按下面跑一遍，然后用应用内的「反馈」按钮告诉我们体感。

---

## 1. 安装（Apple Silicon / M 系列 Mac）

> 这是未签名的内测包，仅支持 Apple Silicon。

1. 打开 `Kairo Code-0.1.0-arm64.dmg`，把 **Kairo Code** 拖进 Applications。
2. 首次启动会被 Gatekeeper 拦“无法验证开发者”。二选一：
   - **右键** App 图标 → **打开** → 在弹窗里再次点「打开」；或
   - 终端执行：`xattr -dr com.apple.quarantine "/Applications/Kairo Code.app"`，再正常双击。
3. 打开后进入 **Settings（⌘,）**，填入你的模型 **API Key / Base URL / Model**（数据只存在本机，不上传）。

---

## 2. 验证场景（请照做）

1. **打开一个你熟悉的代码工程**（⌘O 选择文件夹）。
2. 打开 **Crew**（⌘⇧C），给它一个**会牵动多个模块**的任务，例如：
   - “给 X 功能加一个参数并更新所有调用方”
   - “重构 Y 模块的错误处理”
3. 审一下计划（Plan Gate），点 **Approve & Run**。
4. Crew 跑动时，**留意右侧自动展开的 System Map**：正在改的模块会亮，**改了导出契约/返回结构的模块会出现红色虚线圈**。
5. 跑完后，看底部的 **Comprehension Gate**：
   - 低风险 → 安静绿条“已自动放行”；
   - 高风险 → 一个**具体问题** + 「看那一处 →」按钮（一键打开最该看的文件）+ 通过 / 需要修改。
6. **只用 Gate 的那个问题 + 地图**，判断这次改动能不能信。**给自己计时：能在 60 秒内下结论吗？**

---

## 3. 我们最想知道的 3 件事（请在反馈里回答）

1. Gate 给你的**那一个问题**，问到点子上了吗？还是你直接忽略了它？
2. 「看那一处 →」/ 地图红圈，有没有把你**带到真正该看的地方**？
3. 你**愿意明天再用一次**吗？为什么？

---

## 4. 怎么给反馈

- 应用内右下角 **「反馈」** 按钮（随时可点）：打个分 + 写一句话即可。
- 反馈会存到本机：`~/Library/Application Support/Kairo Code/kairo-feedback.md`
- 如果方便，跑完后把这个文件，以及你工程里的 `.kairo/memory.md`（记录了 Gate 的每次决策）发回给我们——这两份就是我们最看重的数据。
